無料で使える RPA の taskt の使い方を紹介しています

長時間ロボットを動かして、途中でスリープ状態にならないか

たとえば、非常に時間のかかる長いスクリプトを実行していたり、同じ処理を何百回も行わせるためにロボットを動かしていたとします。

このときユーザーから見ると、ロボットが動いている間、パソコンには触れていません。

パソコンに一定時間触れていないと、スリープ状態になってしまい、ロボットが作業を完了できないかと不安になるかもしれません。

しかし、基本的にはこの心配は不要かと思います。

ユーザーから見ると、パソコンには一切触れていません。しかし、パソコン側から見ると、マウスやキーボード操作を行っている状態です。

そのため、ロボット実行中はよほど特殊な状況でない限り、途中でスリープ状態になり作業が完了できなくなる恐れはありません。

ロボット実行中にスリープ状態になる状況とは、下のようなものが考えられます。

  • Pause Script でスリープ状態になるくらい長時間停止させた
  • マウス、キーボード操作をしないスクリプトを長時間実行させた
  • スリープするようにスクリプトを作った
  • 実は作業が完了しており、その後長時間放置した
